# 后续未完成事项参考

本文档用于记录当前没有放进下一阶段开发范围、但后续可能需要继续完善的事项。

当前下一阶段优先级是：

```text
项目费用单图片附件：粘贴上传 + 选择图片上传 + 图片预览
```

以下内容暂不进入下一阶段，只作为后续参考。

## 1. 项目费用单增强

### 1.1 费用单确认后的调整机制

当前费用单确认后不能继续编辑明细，需要作废后重建。

后续可考虑：

1. 已确认费用单允许追加调整记录。
2. 自动生成差额流水。
3. 保留调整日志。
4. 报表按最终有效金额统计。

### 1.2 费用类型字典化

当前费用类型是固定枚举：

1. 材料费
2. 人工费
3. 运输费
4. 安装费
5. 维修返工
6. 其他支出

后续如果实际费用类型变多，可改为字典配置。

注意：费用类型会影响财务分类和项目利润口径，字典化前需要明确每种类型对应的记账规则。

### 1.3 项目费用单 PDF 能力

本次图片附件阶段不做 PDF。

后续可考虑：

1. PDF 上传。
2. PDF 预览。
3. 系统默认工具打开 PDF。
4. 多张图片合并生成 PDF。

## 2. 供应商模块增强

当前供应商只是基础资料，并可被项目费用单选择。

后续可考虑：

1. 供应商对账。
2. 供应商付款统计。
3. 供应商欠款统计。
4. 按供应商查看项目费用单。
5. 供应商附件和票据。
6. 供应商类型字典化。

## 3. 附件系统增强

当前合同附件已经支持图片/PDF、预览、打开、重命名、软删除、文件丢失提示。

后续统一附件体验时可考虑：

1. 附件拖拽排序。
2. 缩略图缓存，避免大图反复转 data URL。
3. 附件物理清理策略。
4. 附件批量删除。
5. 附件批量导出。
6. 附件存储目录迁移工具。

## 4. 工资模块增强

当前工资模块已经支持批次、明细、批量录入、确认、发放、作废和调整已发放明细。

后续可考虑：

1. 扣款拆分为多个可配置扣款项。
2. 社保、公积金、个税参与计算的可配置规则。
3. 工资附件。
4. 工资条导出。
5. 员工工资历史趋势。

## 5. 报表与统计增强

当前报表包括月度收支、项目利润、客户应收、账户余额。

后续可考虑：

1. 供应商对账报表。
2. 项目费用类型分析。
3. 项目材料费用明细报表。
4. 工资统计报表。
5. 附件完整性检查报表。
6. 图表筛选条件持久化。

## 6. 分页与大数据量优化

当前基础资料和财务流水已经是真实服务端分页。

后续可考虑：

1. 工资批次服务端分页。
2. 项目费用单服务端分页。
3. 报表服务端分页。
4. 项目统计列表服务端分页。
5. Select 下拉框远程搜索，避免基础资料很多时一次加载 100 条不够。

## 7. 字典与配置增强

当前已支持部分基础字典。

后续可考虑：

1. 供应商类型字典化。
2. 项目费用类型字典化。
3. 更细的系统设置。
4. 用户可配置默认账户、默认费用类型。
5. 字典项新增/删除能力。

注意：涉及财务口径的字典必须谨慎，不能破坏历史数据和报表统计。

## 8. 权限与安全增强

当前软件是单用户本地密码登录。

后续可考虑：

1. SQLite 文件加密。
2. 操作日志更完整。
3. 多用户和角色。
4. 敏感操作二次确认。
5. 自动锁屏。

## 9. 打包与发布增强

当前已支持本机打包。

后续可考虑：

1. 正式应用图标。
2. macOS 签名和 notarization。
3. Windows 安装包完善。
4. 自动更新。
5. 首次启动向导。

## 10. 暂不建议近期开发的功能

以下功能复杂度较高，建议等核心账务稳定后再考虑：

1. 云端同步。
2. 多人实时协作。
3. OCR 自动识别发票。
4. 发票验真。
5. 银行流水自动导入。
6. 复杂审批流。
