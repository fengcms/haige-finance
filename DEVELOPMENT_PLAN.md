# 小公司本地账务管理系统开发任务规划

## 1. 开发目标

本项目需要开发一款本地桌面账务管理软件，用于管理小型公司的客户、客户项目、合同、员工、账户、收支流水、报表和数据备份。

第一版目标不是追求复杂功能，而是先完成可稳定使用的 MVP，确保以下核心流程可用：

1. 客户可以录入和维护。
2. 客户项目可以录入和维护。
3. 合同金额可以录入并关联客户项目。
4. 员工可以录入和维护。
5. 账户可以录入和维护。
6. 收支分类可以录入和维护。
7. 每一笔收入和支出可以录入。
8. 财务流水可以关联客户、项目、员工、账户和分类。
9. 系统可以自动计算月度收支。
10. 系统可以自动计算客户项目的合同金额、已收款、已支出、应收款、当前毛利和预计毛利。
11. 系统可以统计员工工资。
12. 系统可以统计账户余额。
13. 系统可以导出 Excel。
14. 系统可以本地备份数据。

第一版采用以下技术栈：

```text
Electron

TypeScript
Vite

better-sqlite3
Drizzle ORM
dayjs
```

### 2.1 技术选择要求

1. 使用 Electron 开发本地桌面应用。
2. 使用 React + TypeScript 开发前端界面。
3. 使用 SQLite 保存本地数据。
4. 使用 better-sqlite3 操作 SQLite。
5. 使用 Drizzle ORM 管理数据库 schema 和基础查询。
6. 使用 Ant Design 构建后台管理类界面。
7. 使用 exceljs 实现 Excel 导出。
8. 使用 dayjs 处理日期。
9. 使用 zod 进行表单和接口数据校验。
10. 使用 react-hook-form 管理复杂表单。
11. 使用 zustand 做轻量状态管理。
12. 使用 recharts 做简单报表图表。

## 3. 项目结构要求

项目应采用清晰分层结构，前端页面不得直接操作数据库。

推荐目录结构：

```text
src/
├── main/
│   ├── db/
│   │   ├── index.ts
│   │   ├── schema.ts
│   │   ├── migrate.ts
│   │   └── seed.ts
│   ├── repositories/
│   ├── services/
│   ├── ipc/
│   ├── export/
│   ├── backup/
│   └── main.ts
│
├── preload/
│   └── index.ts
│
├── renderer/
│   ├── App.tsx
│   ├── main.tsx
│   ├── routes/
│   ├── layouts/
│   ├── pages/
│   ├── components/
│   ├── forms/
│   ├── api/
│   ├── stores/
│   └── styles/
│
├── shared/
│   ├── types/
│   ├── constants/
│   ├── schemas/
│   └── utils/
```

### 3.1 分层要求

系统必须分为以下层：

```text
React 页面层
↓
前端 API 调用层
↓
Electron IPC 层
↓
Service 业务逻辑层
↓
Repository 数据访问层
↓
SQLite 数据库
```

要求：

1. React 页面层只负责展示、表单、交互。
2. 前端不得直接访问 SQLite。
3. IPC 层只负责桥接前后端调用。
4. Service 层负责业务规则和财务计算。
5. Repository 层负责数据库增删改查。
6. 报表计算逻辑应集中在 Service 层或专门的 ReportService 中。
7. 数据库 schema、共享类型、枚举常量应统一维护，避免重复定义。

## 阶段五：报表与对账模块

### 目标

完成第一版核心统计报表。

### 任务

1. 首页仪表盘

   * 未结清项目数量

2. 月度收支报表

   * 按月份统计收入
   * 按月份统计支出
   * 计算月度盈余
   * 按分类展示收入支出

3. 客户项目报表

   * 合同金额
   * 已收款
   * 已支出
   * 应收款
   * 当前毛利
   * 预计毛利
   * 收入明细
   * 支出明细

4. 应收账款报表

   * 展示所有未收齐项目
   * 展示客户、项目、合同金额、已收款、应收款
   * 支持按客户和项目状态筛选

5. 员工工资报表

   * 按月份统计员工工资
   * 按员工查看工资明细

6. 账户余额报表

   * 按账户统计收入
   * 按账户统计支出
   * 计算账户当前余额

7. 股东往来报表

   * 统计股东注资
   * 统计股东代收
   * 统计股东代付
   * 统计股东借款

### 核心公式

```text
项目已收款 = 项目下所有状态正常且资金性质为客户收款的收入流水金额之和

项目已支出 = 项目下所有状态正常且属于项目支出的支出流水金额之和

项目应收款 = 项目合同金额 - 项目已收款

项目当前毛利 = 项目已收款 - 项目已支出

项目预计毛利 = 项目合同金额 - 项目已支出

账户当前余额 = 账户初始余额 + 账户正常收入总额 - 账户正常支出总额

月度盈余 = 月度收入总额 - 月度支出总额
```

### 验收标准

1. 首页可以显示核心经营数据。
2. 月度报表可以准确统计收入、支出、盈余。
3. 项目报表可以准确统计合同金额、已收款、已支出、应收款、毛利。
4. 应收账款报表可以准确显示未收齐项目。
5. 员工工资报表可以准确统计工资。
6. 账户余额报表可以准确计算余额。
7. 作废流水不参与统计。
8. 股东注资、股东代收、股东代付、借款不得错误混入客户项目收入或普通费用。
