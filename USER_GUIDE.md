# 用户操作手册草稿

本文档面向日常使用者，说明第一版 MVP 的主要操作流程。

## 1. 基础资料录入顺序

建议按以下顺序录入：

1. 客户管理
2. 客户项目
3. 合同管理
4. 员工管理
5. 供应商管理
6. 账户管理
7. 系统设置中的收支分类
8. 财务管理中的收支流水
9. 工资管理中的工资批次和工资明细

先录基础资料，再录流水，后续项目统计和报表才能正确关联。

## 2. 客户管理

进入“客户管理”页面，可以新增、编辑、删除客户。

客户删除为软删除，不会从数据库中物理移除。

## 3. 客户项目

进入“客户项目”页面，可以维护客户对应的项目。

项目必须选择所属客户。

项目列表会显示：

1. 合同金额
2. 已收款
3. 已支出
4. 应收款
5. 当前毛利
6. 预计毛利
7. 收款状态

点击“统计”可以查看项目详情，包括合同列表和相关流水。

## 4. 项目收支

进入“项目收支”页面，可以围绕某个项目快速录入收款和支出。

建议先选择项目，再使用快捷按钮：

1. 项目收款
2. 材料支出
3. 人工支出
4. 运输费
5. 安装费
6. 维修返工
7. 其他支出

项目收支保存后，会自动生成财务流水，并自动关联当前客户和项目。

项目收款会影响项目应收款，不直接影响项目利润。

材料支出、人工支出和其他项目支出会影响项目利润。

这些流水仍然可以在“财务管理”中查看。

页面会按项目收款、材料费、人工费、运输费、安装费、维修返工和其他支出显示分类小计，也可以按类型筛选项目流水。

项目费用需要多条明细时，可以使用“项目费用单”：

1. 点击“新增费用单”，选择费用类型、供应商、费用日期和计划付款账户。
2. 进入费用单详情后，点击“新增一行”，在表格中直接录入名称、规格、数量、单位、单价和备注。
3. 可以连续新增多行，确认无误后点击“一次性保存”。
4. 明细金额会汇总到费用单合计。
5. 草稿费用单可以继续新增明细或删除草稿。
6. 点击“确认生成流水”后，系统会生成一条项目支出财务流水，并参与项目利润统计。
7. 已确认费用单可以作废，系统会联动作废对应财务流水，作废后不再参与项目统计。

费用单现在支持图片附件：

1. 在费用单详情中找到“附件图片”区域。
2. 点击粘贴区域后，可以直接粘贴系统截图、微信截图或 QQ 截图。
3. 也可以点击“选择图片”，从本地选择 JPG、PNG、WEBP 图片。
4. 上传后可以在附件列表中点击“预览”查看大图。
5. 可以删除已上传图片，删除为软删除，本地文件会保留。
6. 已作废费用单只允许查看附件，不允许继续上传。

当前费用单附件暂不支持 PDF、图片转 PDF、OCR 或拖拽排序。

费用单状态显示来自“系统设置 / 字典设置 / 项目费用单状态”。可以在字典设置中修改“草稿、已确认、已作废”的显示名称、排序和启停状态。

## 5. 供应商管理

进入“供应商管理”页面，可以维护材料商、人工班组、运输、安装和其他供应商。

供应商信息包括：

1. 供应商名称
2. 联系人
3. 联系电话
4. 地址
5. 供应商类型
6. 状态
7. 备注

供应商删除为软删除，不会从数据库中物理移除。

第一版供应商管理只做基础资料维护。项目费用单可以选择供应商，后续再扩展供应商对账。

## 6. 合同管理

进入“合同管理”页面，可以录入客户项目合同。

合同金额在页面输入为元，系统内部保存为整数分。

合同必须关联客户和项目，且项目必须属于所选客户。

点击合同列表中的“附件”可以维护合同附件。

当前支持：

1. 上传 JPG、PNG、WEBP 图片。
2. 上传 PDF 文件。
3. 对图片和 PDF 附件手动上移、下移排序。
4. 将当前合同下的图片按排序生成一个 PDF 文件。
5. 在系统内预览图片和 PDF。
6. 使用系统默认工具打开已上传或已生成的 PDF。
7. 重命名附件显示名称。
8. 查看附件大小、创建时间和本地文件状态。

附件删除为软删除，文件会保留在本地数据目录中，列表中不再显示。

如果附件对应的本地文件被手动删除，系统会显示“本地文件丢失”，并禁用预览和打开操作。

## 7. 财务管理

进入“财务管理”页面，可以录入收入和支出。

流水需要选择：

1. 发生日期
2. 收支分类
3. 方向
4. 金额
5. 账户
6. 资金性质

可选关联：

1. 客户
2. 项目
3. 员工

选择收支分类后，系统会自动带出方向、资金性质、是否影响应收、是否影响项目利润。

## 8. 作废流水

财务流水不得物理删除。

如果一笔流水录错，优先使用“作废”，并填写作废原因。

作废流水不会参与：

1. 账户余额
2. 项目统计
3. 报表统计

## 9. 报表对账

进入“报表对账”页面，可以查看：

1. 月度收支表
2. 项目利润表
3. 客户应收表
4. 账户余额表

报表金额显示为元。

## 10. 工资管理

进入“工资管理”页面，可以维护员工工资。

建议流程：

1. 新增工资批次。
2. 在工资批次详情中新增员工工资明细，或使用“批量录入”一次录入多名员工。
3. 确认工资批次。
4. 发放工资，选择发放日期和发放账户。

工资明细支持录入：

1. 基本工资
2. 全勤
3. 话补
4. 奖金
5. 提成
6. 扣款
7. 社保
8. 公积金
9. 个税

当前计算口径：

```text
应发工资 = 基本工资 + 全勤 + 话补 + 奖金 + 提成
实发工资 = 应发工资 - 扣款
```

社保、公积金、个税当前只记录，不参与实发工资计算。

发放工资后，系统会自动生成一条“人工工资”财务支出流水。该流水不关联客户和项目，不影响项目利润，但会影响账户余额和月度支出。

已发放工资可以编辑已有工资明细，系统会同步调整关联财务流水金额，并记录操作日志。

作废已发放工资时，系统会联动作废对应财务流水，并记录作废原因。

批量录入时，顶部会显示员工标签。点击员工标签后，下方表格会增加该员工的一行工资输入项。已在当前批次录入过工资的员工会显示“已录”，不能重复选择。

批量录入支持一次性保存多名员工。保存时系统会统一校验，如果某一行不合法，整批数据不会保存一半。

## 11. 首页仪表盘

首页展示当前经营概览：

1. 本月收入
2. 本月支出
3. 本月收支差额
4. 账户总余额
5. 项目应收合计
6. 项目预计毛利合计

## 12. 数据备份

进入“系统设置”页面的“数据备份”栏目，点击“备份数据库”。

系统会生成 SQLite 备份文件，并提示备份路径。

建议在大量录账后、月底对账后、导出报表前手动备份。

## 13. 数据恢复

进入“系统设置”页面的“数据备份”栏目，点击“恢复数据库”。

恢复流程：

1. 系统会提示确认恢复风险。
2. 选择一个 SQLite 数据库备份文件。
3. 系统先自动备份当前数据库。
4. 系统校验恢复文件是否包含必要账务表。
5. 校验通过后替换当前数据库。
6. 恢复成功后建议重启应用。

如果恢复后发现数据不对，可以点击“撤销最近一次恢复”。

撤销恢复时，系统也会先备份当前数据库，然后恢复到上一次恢复前的状态。

## 14. Excel 导出

进入“系统设置”页面的“数据备份”栏目，点击“导出 Excel”。

导出内容包括：

1. 客户
2. 项目
3. 合同
4. 财务流水
5. 月度收支表
6. 项目利润表
7. 客户应收表
8. 账户余额表

Excel 中金额字段以元展示。

## 15. 界面设置

进入“系统设置”页面的“界面设置”栏目，可以配置默认每页条数。

当前可选：

1. 10 条 / 页
2. 20 条 / 页
3. 50 条 / 页
4. 100 条 / 页

默认值为 20 条 / 页。

保存后，新打开或刷新后的主要列表会使用该默认值。表格右下角仍可以临时切换当前列表的每页条数。

## 16. 字典设置

进入“系统设置”页面的“字典设置”栏目，可以维护部分基础字典的显示名称、排序和启停状态。

当前支持：

1. 客户状态
2. 项目状态
3. 项目类型
4. 合同状态
5. 员工状态
6. 账户类型
7. 账户状态
8. 项目费用单状态

字典编码由系统使用，不能在页面中修改。资金性质、流水方向、流水状态等会影响报表和财务口径的字段，当前不开放字典编辑。

停用字典项后，新建或编辑业务数据时下拉框不再显示该项；已有历史数据仍会保留原编码，并尽量按字典名称展示。

## 17. 登录与安全设置

首次打开软件时，需要设置本地管理员密码。

后续每次打开软件，需要输入密码后才能进入系统。

当前用户固定为：

```text
admin
```

进入系统后，可以在右上角点击“退出登录”，返回登录页面。

如需修改密码，进入“系统设置 → 安全设置”，输入旧密码和新密码后保存。

密码修改成功后，系统会要求重新登录。

忘记密码时，请查看：

```text
PASSWORD_RESET_GUIDE.md
```

## 18. 暂不支持的功能

当前版本暂不支持：

1. 附件拖拽排序
2. 多账套
3. 权限系统
4. 云同步
5. 自动更新
6. 正式签名安装包
7. 工资独立报表
8. 工资条导出
