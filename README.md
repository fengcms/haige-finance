# 小公司本地账务管理系统需求文档

## 1. 项目概述

本项目需要开发一款简单、稳定、易维护的本地软件，用于管理小型公司的客户资料、员工资料、客户合同、收支流水及财务对账。

系统应重点解决当前使用在线表格记账时存在的数据分散、对账困难、客户收支不清、合同应收款无法自动统计、股东代收代付容易混乱等问题。

系统的核心目标是：

1. 管理客户基础信息。
2. 管理客户项目及合同金额。
3. 管理员工基础信息。
4. 记录每一笔收入和支出。
5. 将客户相关收支绑定到具体客户项目。
6. 自动统计客户项目的已收款、已支出、应收款和利润情况。
7. 自动统计月度收入、支出和账户盈余。
8. 区分公司账户、客户收入、股东注资、股东代收、股东代付、借款等不同资金性质。
9. 支持后续查账、对账、备份和导出。

本系统不是通用记账本，而是面向小型装修、服务类公司的本地经营账务管理系统。

---

## 2. 核心业务原则

系统中每一笔资金流水都必须能够回答以下问题：

1. 这笔钱是收入还是支出？
2. 这笔钱的金额是多少？
3. 这笔钱发生在什么日期？
4. 这笔钱属于哪个账户？
5. 这笔钱是否进入公司账户？
6. 这笔钱是否关联客户？
7. 这笔钱是否关联具体客户项目？
8. 这笔钱是否关联员工？
9. 这笔钱属于什么收支分类？
10. 这笔钱是否属于股东注资、代收、代付或借款？
11. 这笔钱是否影响客户合同应收款？
12. 这笔钱是否影响项目利润？

系统必须明确区分以下概念：

1. 股东注资不等于客户收入。
2. 客户收款不等于普通营业外收入。
3. 代收款不等于已经进入公司账户。
4. 代付款不等于公司账户直接支出。
5. 借款不等于普通费用。
6. 合同金额不等于已收款。
7. 已收款不等于项目利润。

## 4. 客户管理要求

### 4.1 客户基础信息

系统应支持维护客户基础资料。

客户信息至少包含以下字段：

| 字段   | 要求                        |
| ---- | ------------------------- |
| 客户姓名 | 必填                        |
| 联系电话 | 必填或选填，由系统配置决定             |
| 客户地址 | 选填                        |
| 小区名称 | 选填                        |
| 房号   | 选填                        |
| 添加日期 | 必填，默认当前日期                 |
| 客户备注 | 选填                        |
| 客户状态 | 可选，如潜在客户、已签约、施工中、已完工、无效客户 |

### 4.2 客户列表

客户列表应支持：

1. 查看所有客户。
2. 按客户姓名搜索。
3. 按电话搜索。
4. 按小区搜索。
5. 按添加日期筛选。
6. 查看客户关联的项目数量。
7. 查看客户是否存在未结清合同。

### 4.3 客户详情

客户详情页应显示：

1. 客户基础信息。
2. 客户关联的所有项目。
3. 客户关联的所有合同。
4. 客户关联的所有收入流水。
5. 客户关联的所有支出流水。
6. 客户累计合同金额。
7. 客户累计已收款。
8. 客户累计应收款。
9. 客户累计项目支出。
10. 客户累计项目毛利。

## 5. 客户项目管理要求

系统必须设置“客户项目”概念，不能只依赖客户姓名进行财务统计。

一个客户可以拥有一个或多个项目。

客户项目至少包含以下字段：

| 字段        | 要求                          |
| --------- | --------------------------- |
| 项目名称      | 必填，例如“君安华庭 11-502 装修项目”     |
| 所属客户      | 必填                          |
| 小区名称      | 必填或选填                       |
| 房号 / 施工地址 | 必填或选填                       |
| 项目类型      | 可选，如装修、橱柜、维修、其他             |
| 项目状态      | 可选，如待签约、已签约、施工中、已完工、已结清、已作废 |
| 创建日期      | 必填                          |
| 备注        | 选填                          |

### 5.1 项目详情

项目详情页应显示：

1. 项目基础信息。
2. 所属客户信息。
3. 项目合同信息。
4. 项目收入流水。
5. 项目支出流水。
6. 项目已收款。
7. 项目已支出。
8. 项目应收款。
9. 项目当前毛利。
10. 项目预计毛利。

### 5.2 项目统计公式

系统应按照以下公式自动计算项目数据：

```text
项目已收款 = 该项目下所有客户收款之和

项目已支出 = 该项目下所有项目相关支出之和

项目应收款 = 项目合同金额 - 项目已收款

项目当前毛利 = 项目已收款 - 项目已支出

项目预计毛利 = 项目合同金额 - 项目已支出
```

## 6. 客户合同管理要求

客户合同管理用于维护客户项目的合同信息，重点是合同金额。

### 6.1 合同基础信息

合同信息至少包含以下字段：

| 字段   | 要求         |
| ---- | ---------- |
| 合同编号 | 可自动生成      |
| 所属客户 | 必填         |
| 所属项目 | 必填         |
| 合同名称 | 必填         |
| 合同金额 | 必填         |
| 签约日期 | 选填         |
| 合同状态 | 必填         |
| 合同备注 | 选填         |
| 合同附件 | 选填，可后续版本支持 |

合同状态至少包括：

1. 未签约
2. 已签约
3. 进行中
4. 已完工
5. 已结清
6. 已作废

### 6.2 合同金额与应收款

系统必须基于合同金额和项目收款自动计算应收账款。

```text
合同应收款 = 合同金额 - 项目已收款
```

当项目已收款大于或等于合同金额时，系统应能识别该合同为已收齐或超收状态。

### 6.3 合同附件

系统应预留合同附件功能。

附件类型可包括：

1. PDF
2. Word
3. 图片
4. 扫描件

第一版可以暂不实现附件上传，但数据库和页面设计应预留扩展空间。

## 7. 员工管理要求

员工管理用于维护员工基础资料，并在财务支出中记录工资归属，避免工资支出记错人员。

### 7.1 员工基础信息

员工信息至少包含以下字段：

| 字段   | 要求        |
| ---- | --------- |
| 员工姓名 | 必填        |
| 联系电话 | 选填        |
| 岗位   | 选填        |
| 入职日期 | 选填        |
| 员工状态 | 必填，如在职、离职 |
| 备注   | 选填        |

### 7.2 员工与工资支出

当财务流水的支出分类为“工资”时，系统应支持选择对应员工。

系统应支持按员工统计工资支出：

1. 某员工累计工资。
2. 某员工某月份工资。
3. 全部员工某月份工资总额。
4. 工资支出明细。

## 8. 财务流水管理要求

财务流水管理是系统核心模块。

系统必须支持录入每一笔收入和每一笔支出。

### 8.1 流水基础字段

每一笔财务流水至少包含以下字段：

| 字段       | 要求       |
| -------- | -------- |
| 流水类型     | 必填，收入或支出 |
| 金额       | 必填       |
| 发生日期     | 必填       |
| 账户       | 必填       |
| 收支分类     | 必填       |
| 客户       | 选填       |
| 客户项目     | 选填       |
| 员工       | 工资支出时必填  |
| 用途说明     | 选填       |
| 经办人      | 选填       |
| 是否进入公司账户 | 必填       |
| 资金性质     | 必填       |
| 备注       | 选填       |
| 创建时间     | 系统自动记录   |
| 更新时间     | 系统自动记录   |

### 8.2 流水类型

流水类型只分为：

1. 收入
2. 支出

### 8.3 资金性质

资金性质必须进一步区分，至少包括：

#### 收入类

1. 客户收款
2. 股东注资
3. 股东代收
4. 借款收入
5. 其他收入

#### 支出类

1. 项目支出
2. 工资支出
3. 房租支出
4. 广告推广支出
5. 办公支出
6. 股东代付
7. 借款支出
8. 其他支出

### 8.4 收入录入要求

收入录入时，应支持以下情况：

1. 客户订金。
2. 客户二期款。
3. 客户尾款。
4. 客户其他收款。
5. 股东注资。
6. 股东代收客户款。
7. 其他收入。

当收入属于客户收款时，必须关联客户项目。

当收入属于股东注资时，不应计入客户项目收入。

当收入属于股东代收时，应记录代收人，并标记该款项是否已经转入公司账户。

### 8.5 支出录入要求

支出录入时，应支持以下情况：

1. 项目材料费。
2. 项目人工费。
3. 员工工资。
4. 办公室房租。
5. 广告推广费。
6. 财务代账费。
7. 办公设备费。
8. 股东代付。
9. 借款。
10. 其他支出。

当支出属于项目支出时，必须关联客户项目。

当支出属于工资时，应支持选择对应员工。

当支出属于股东代付时，应记录代付人，并纳入股东往来统计。

## 9. 账户管理要求

系统必须支持账户管理，不能只记录“渠道”文字。

### 9.1 账户类型

系统至少应支持以下账户类型：

1. 公司银行账户
2. 公司微信账户
3. 公司支付宝账户
4. 股东个人账户
5. 现金账户
6. 其他账户

### 9.2 账户字段

账户至少包含以下字段：

| 字段     | 要求            |
| ------ | ------------- |
| 账户名称   | 必填，例如“公司建行账户” |
| 账户类型   | 必填            |
| 所属人    | 选填，如刘总、钱总     |
| 是否公司账户 | 必填            |
| 初始余额   | 选填            |
| 账户状态   | 必填，如启用、停用     |
| 备注     | 选填            |

### 9.3 账户余额计算

系统应根据流水自动计算账户余额。

```text
账户当前余额 = 账户初始余额 + 该账户收入总额 - 该账户支出总额
```

系统应支持查看每个账户的收入、支出和余额。

系统应支持维护收入分类和支出分类。

### 10.1 收入分类

系统默认收入分类至少包括：

1. 股东注资
2. 客户订金
3. 客户阶段款
4. 客户尾款
5. 客户其他收款
6. 借款收入
7. 其他收入

### 10.2 支出分类

系统默认支出分类至少包括：

1. 房租物业
2. 广告推广
3. 员工工资
4. 项目材料
5. 项目人工
6. 办公设备
7. 财务服务
8. 股东借款
9. 股东代付
10. 其他支出

### 10.3 分类要求

分类应支持：

1. 新增。
2. 编辑。
3. 停用。
4. 排序。
5. 区分收入分类和支出分类。

已被流水使用的分类不应允许直接删除，只能停用。

## 11. 报表与对账要求

系统必须提供财务报表与对账功能。

### 11.1 首页仪表盘

首页应展示以下核心数据：

1. 本月收入。
2. 本月支出。
3. 本月盈余。
4. 本月客户收款。
5. 本月项目支出。
6. 当前应收账款总额。
7. 当前未结清项目数量。
8. 最近财务流水。

### 11.2 月度收支报表

系统应支持按月份查看：

1. 月度总收入。
2. 月度总支出。
3. 月度盈余。
4. 客户收款金额。
5. 股东注资金额。
6. 项目支出金额。
7. 工资支出金额。
8. 房租支出金额。
9. 广告推广支出金额。
10. 其他支出金额。

### 11.3 客户项目报表

系统应支持按客户项目查看：

1. 合同金额。
2. 已收款。
3. 已支出。
4. 应收款。
5. 当前毛利。
6. 预计毛利。
7. 收入明细。
8. 支出明细。

### 11.4 应收账款报表

系统应支持查看所有未收齐款项的项目。

应收账款报表至少包含：

| 字段     |
| ------ |
| 客户     |
| 项目     |
| 合同金额   |
| 已收款    |
| 应收款    |
| 项目状态   |
| 最近收款日期 |
| 备注     |

### 11.5 员工工资报表

系统应支持按员工和月份统计工资支出。

报表至少包含：

1. 员工姓名。
2. 月份。
3. 工资支出总额。
4. 工资支出明细。

### 11.6 股东往来报表

系统应支持统计股东相关资金往来。

股东往来至少包括：

1. 股东注资。
2. 股东代收。
3. 股东代付。
4. 股东借款。
5. 公司应付股东金额。
6. 股东应转入公司金额。

### 11.7 账户余额报表

系统应支持按账户查看：

1. 账户名称。
2. 初始余额。
3. 收入总额。
4. 支出总额。
5. 当前余额。
6. 流水明细。

## 12. 数据导入与导出要求

### 12.1 数据导入

系统应预留数据导入能力。

导入方式可包括：

1. Excel 导入。

1. 客户列表。

3. 合同列表。
4. 财务流水。
5. 月度收支报表。

## 13. 数据备份要求

由于系统为本地软件，必须重视数据安全。

系统应支持：

1. 手动备份。
2. 自动备份。
3. 恢复备份。
4. 导出完整数据。
5. 备份文件保存到用户指定目录。

自动备份建议至少支持每日备份。

## 14. 操作安全要求

### 14.1 财务流水不允许直接物理删除

财务流水一旦产生，不应允许直接物理删除。

系统应支持：

1. 作废流水。
2. 修改流水。
3. 查看修改记录。
4. 标记异常流水。

### 14.2 操作日志

系统应记录关键操作日志。

日志至少包含：

1. 操作人。
2. 操作时间。
3. 操作类型。
4. 操作对象。
5. 修改前内容。
6. 修改后内容。

第一版可以简化操作日志，但数据库设计应预留扩展空间。

---

## 15. 权限要求

系统应预留权限管理能力。

角色至少包括：

1. 管理员
2. 财务人员
3. 普通员工

权限示例：

| 角色   | 权限                        |
| ---- | ------------------------- |
| 管理员  | 可查看、添加、修改、导出全部数据          |
| 财务人员 | 可管理财务流水、客户合同、报表           |
| 普通员工 | 可查看或录入客户信息，但不能删除或修改财务核心数据 |

第一版如果只在本地单人使用，可以暂不实现复杂权限，但应预留用户和角色字段。

├── 项目列表

## 18. 核心计算公式

系统必须内置以下核心计算逻辑：

```text
项目已收款 = 项目下所有客户收款收入之和

项目已支出 = 项目下所有项目相关支出之和

项目应收款 = 项目合同金额 - 项目已收款

项目当前毛利 = 项目已收款 - 项目已支出

项目预计毛利 = 项目合同金额 - 项目已支出

客户累计合同金额 = 客户下所有项目合同金额之和

客户累计已收款 = 客户下所有项目已收款之和

客户累计应收款 = 客户累计合同金额 - 客户累计已收款

账户当前余额 = 账户初始余额 + 账户收入总额 - 账户支出总额

月度盈余 = 月度收入总额 - 月度支出总额
```

## 19. 数据准确性要求

系统必须保证以下数据准确性：

1. 金额必须使用数字类型存储，不允许只存文本。
2. 日期必须使用标准日期格式存储。
3. 客户项目相关流水必须绑定项目，不允许只在备注里填写项目名称。
4. 工资支出必须能够绑定员工。
5. 合同金额必须能够参与应收账款计算。
6. 已作废流水不应参与正常统计。
7. 股东注资、股东代收、股东代付不得混入普通客户收入或普通公司费用。
8. 财务报表数据必须来源于流水记录自动汇总，不应手工填写汇总数。
9. 已被使用的客户、项目、员工、账户、分类不应允许直接删除。
10. 系统应尽量避免重复录入同一客户、同一项目、同一笔流水。

## 20. 本地软件要求

系统应作为本地软件运行，要求：

1. 不依赖在线表格软件。
2. 数据保存在本地。
3. 支持本地备份。
4. 支持导出 Excel。
5. 界面简单清晰。
6. 操作流程适合非专业财务人员使用。
7. 后续应具备扩展为多人使用或云端同步的可能性。

---

## 21. 项目最终目标

本系统最终应帮助公司实现以下管理目标：

1. 清楚知道每个月收入多少、支出多少、盈余多少。
2. 清楚知道每个客户项目收了多少钱、花了多少钱、还差多少钱没收。
3. 清楚知道每个项目当前是否盈利。
4. 清楚知道公司账户、微信、支付宝、股东个人代收代付之间的资金关系。
5. 清楚知道员工工资支出。
6. 清楚知道股东注资、借款、代收、代付情况。
7. 能够方便地查账、对账、导出和备份。
8. 减少手工表格记账造成的漏记、错记、重复记账和对账困难。
