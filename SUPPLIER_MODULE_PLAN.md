# 供应商管理模块开发计划

## 1. 模块定位

供应商属于基础资料，用于后续项目费用单、采购、票据和供应商对账。

第一版只做供应商基础档案，不做采购单、不做应付账款、不做供应商统计。

## 2. 第一版字段

```text
供应商名称
联系人
联系电话
地址
供应商类型
状态
备注
```

供应商类型：

```text
材料商
人工班组
运输
安装
其他
```

状态：

```text
启用
停用
```

## 3. 第一版功能

1. 新增“供应商管理”菜单。
2. 供应商列表。
3. 新增供应商。
4. 编辑供应商。
5. 删除供应商，使用软删除。
6. 搜索供应商名称、联系人、电话、地址。
7. shared 增加供应商类型、状态、zod schema。
8. SQLite 增加 `suppliers` 表。
9. Drizzle schema 增加供应商表定义。
10. Service / Repository / IPC / preload / renderer API 按现有基础资料架构接入。
11. smoke test 覆盖新增、编辑、软删除。

## 4. 后续扩展

1. 项目费用单关联供应商。
2. 供应商采购记录。
3. 供应商付款记录。
4. 供应商欠款和对账。
5. 供应商附件和票据。

## 5. 验收标准

1. 可以进入“供应商管理”页面。
2. 可以新增供应商。
3. 可以编辑供应商。
4. 可以软删除供应商。
5. 可以按关键词搜索供应商。
6. `pnpm supplier:smoke-test` 通过。
7. `pnpm verify` 通过。
